# Plan 02: Usage Dashboard UI

## Objective

Replace the usage placeholder page with a dashboard showing per-client token usage table with period selector, estimated costs, and rate limit display. Update the main dashboard page to show live stats from the API.

## Tasks

### Task 1: Build the usage page

<read_first>
- portal/app/portal/usage/page.tsx
- portal/app/api/usage/route.ts
- portal/app/api/usage/limits/route.ts
- portal/app/globals.css
</read_first>

<action>
Replace `portal/app/portal/usage/page.tsx` with a client component that:

1. Fetches `/api/usage?period=week` on load
2. Period selector buttons (Day / Week / Month) that refetch
3. Table showing: client name, requests, input tokens, output tokens, cache tokens, avg latency, estimated cost
4. Rate limits section fetching `/api/usage/limits`
5. Dark theme consistent with portal

Key elements:
- Period selector as button group
- Usage table with formatted numbers (toLocaleString)
- Cost column with $ prefix
- Rate limits card showing remaining/total for requests and tokens
- "No data" state when no logs exist
</action>

<acceptance_criteria>
- `portal/app/portal/usage/page.tsx` contains `'use client'`
- `portal/app/portal/usage/page.tsx` contains `fetch('/api/usage`
- `portal/app/portal/usage/page.tsx` contains `fetch('/api/usage/limits`
- `portal/app/portal/usage/page.tsx` contains `Day` and `Week` and `Month`
- `portal/app/portal/usage/page.tsx` contains `estimated_cost`
- `portal/app/portal/usage/page.tsx` contains `Remaining`
</acceptance_criteria>
