# Plan 03: Login Page & Portal Dashboard Shell

## Objective

Build the login page UI, the authenticated portal layout with left sidebar navigation, the dashboard skeleton page, and placeholder pages for Tokens, OAuth, Usage, and Logs. After this plan, an admin can log in via the browser and see the full portal shell with working navigation.

## Tasks

- portal/lib/auth.ts
- portal/app/globals.css
- .planning/phases/01-foundation-admin-auth/01-CONTEXT.md
</read_first>

<action>
Create `portal/app/login/page.tsx`:

A dark-themed login page with:
- Centered card on dark background
- "CC Proxy" title with monospace accent
- Password input field (type="password")
- "Sign in" button
- Error message display area
- Client-side form submission to `POST /api/auth/login`
- On success: redirect to `/portal` using `window.location.href = '/portal'`
- On error: show error message from API response

Mark as client component with `'use client'` directive.

Use Tailwind classes for styling:
- Background: `bg-[hsl(var(--background))]`
- Card: `bg-[hsl(var(--card))] border border-[hsl(var(--border))] rounded-lg p-8`
- Input: `bg-[hsl(var(--input))] border border-[hsl(var(--border))] rounded-md px-3 py-2 text-sm`
- Button: `bg-[hsl(var(--primary))] text-white rounded-md px-4 py-2 text-sm font-medium`
- Error: `text-[hsl(var(--destructive))] text-sm`
- Title: `text-2xl font-bold font-mono`

The form should have:
- `<input name="password" type="password" placeholder="Admin password" autoFocus />`
- `<button type="submit">Sign in</button>`
- Loading state on the button during submission
</action>

## Verification

After all tasks complete:
1. Navigate to `http://localhost:8443/login` → see login form with "CC Proxy" title and password input
2. Enter correct password → redirected to `/portal` with sidebar visible
3. Sidebar shows 5 navigation items with icons
4. Click each nav item → corresponding placeholder page loads
5. Dashboard shows 3 stat cards with "—" values
6. Click Logout → redirected to `/login`
7. Navigate to `/portal` without session → redirected to `/login`
8. Refresh browser on `/portal` → stays on portal (session persists)
