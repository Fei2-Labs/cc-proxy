# Plan: Docker Build & Compose

## Goal
Update Dockerfile for multi-stage build producing a single image with proxy + portal, update docker-compose.yml for Dokploy, create .dockerignore.

## Tasks

### Task 1: Add Next.js standalone output
Update `portal/next.config.mjs` to add `output: 'standalone'` for optimized Docker builds.

### Task 2: Create .dockerignore
Exclude node_modules, .next, .planning, .git, .playwright-mcp, data, certs, dist, and other dev files.

### Task 3: Rewrite Dockerfile
Multi-stage build:
- **Stage 1 (builder):** node:22-slim, install pnpm, copy package.json + pnpm-lock.yaml, install deps, copy source, run `pnpm run build` (tsc + next build)
- **Stage 2 (runner):** node:22-slim, copy compiled dist/, Next.js standalone output, static assets, config.example.yaml, create data directory
- Entry point: `node dist/server.js`
- Expose port 8443

### Task 4: Update docker-compose.yml
- Add `data` volume for SQLite persistence
- Add environment variables: `NODE_ENV`, `ADMIN_PASSWORD`, `PORTAL_DATA_DIR`
- Keep config.yaml mount
- Remove certs mount (not needed for Dokploy reverse proxy setup)

### Task 5: Verify build
Run `docker build` to confirm the image builds successfully.

## Success Criteria
1. `docker build` produces a working image
2. docker-compose.yml has correct volumes, ports, and env vars for Dokploy
3. Image uses pnpm and builds both proxy and portal
